# Persona Presence & Editorial

**Status**: active (started 2026-07-12) · **Branches**: `feat/persona-units` (appearance), `feat/writing-lab-v2` (editorial)

Two coupled threads off `dev`, both descending from the 07-08 persona system. One pipe: **author (writing lab) → generate (seeded `lib/seed`) → present (appearance)**. Determinism is the rail under both — new fields at stream tails, never re-roll existing output.

## Thread A — Appearance: tenant-region highlighting + night lighting

**Goal**: make a selected resident/business *visible in the 3D world* by lighting their parcel, and give the ambient city believable per-tenant lighting.

**Model (locked 2026-07-12, validated by prototype)** — a deterministic `tenancyLayout(building, households, businesses, rng) → Region[]`:
- A **region** is a span of bays × floors on one or more faces: unit / floor-band / whole-building, scaling by building size. We know a resident's real floor (`unitFor`); bay/face is newly assigned.
- Commerce is **district-textured**: downtown → multi-floor **department stores** + wide storefronts; main street → a **row of small storefronts**; residential → townhome vertical parcels; company-owned → whole shell.
- Each region carries a **light signature** `{hex, intensity, variance, dark-chance}` — storefront variety is a main/high-street trait; department stores uniform bright white; offices mostly-dark-cool; homes warm-scattered. One function yields both **where** a tenant sits and **how** it's lit.
- Prototype: `public/prototypes/tenancy-layout.html` (Settings → Labs → Tenancy Layout, or open locally; toggles tenant-type / night-lights). Moved from `docs/prototypes/` 2026-07-15 so the app serves it.

**Highlight mechanism** (from feasibility dig): per-selection shader **uniforms** the `cityInstanced` shader compares against physical `(vFaceId, floor(cell.x), floor(cell.y))` — reuses the #87 position-compare + #86 floor-test + emissive glow, no new per-instance attribute (16-attrib ceiling). Whole-building reuses `SelectedBuildingOutline`; falls back to it at distance. Transient selection-state only — never fed to generation.

**Port target**: `lib/seed/tenancyLayout.ts` (+ light signature), then wire uniforms in `cityInstanced.ts` / `InstancedCity.tsx` and selection in `sceneStore.ts`.

## Thread B — Editorial: the lab and app converge

**Goal**: stable searchable IDs; in-app editor mode; slot visibility (which content slots a persona fills) in app + lab; the lab loads real personas from a seed and reviews cards in context; safe write-back.

**Foundation ("Spine 2", plan 2026-07-12)**:
- **Hybrid stable IDs** — source pools stay `string[]`; a committed sidecar `lib/writing/contentIds.ts` maps index → frozen `poolId~ordinal` (generated by `scripts/genContentIds.ts`). Zero churn to the determinism-critical `arr[floor(rng()·len)]` draw. Beats explicit-restructure and content-hash.
- **Provenance** — a non-consuming `pickIdx` (byte-identical to `pick`) captures the final picked index+pool per slot into a WeakMap beside `weaveStates`; `personaProvenance` (forward, lazy) + `buildInverseProvenance` (lab-only). Eager tier (names/profession) via value-lookup.
- **Finding A**: resident *given* names come from `nameCohorts.ts` decade tables, NOT the flat pools the lab shows (those feed only offstage + business `{G}`). Provenance routes names correctly.
- **Finding B**: a "safe" text edit = same length + position + **slot-token signature** (counts of `{N}`/`{lore:*}`/`{kin}` that `fill()` consumes). The write-back guard enforces this and kills the silent `exportPoolAsTs` cut-drop footgun.
- **Determinism gate**: `bun scripts/personaCheck.ts` byte-identical before/after the `pickIdx` instrumentation. `cityGolden` is geometry-only — untouched.

Implementation delegated to an agent on `feat/writing-lab-v2` (2026-07-12); diffs reviewed + gated before integration.

## Decisions
- 2026-07-12 — appearance atom is a **tenant-region span**, not a single window; regions in physical facade coords so spans stay coherent despite the shared-across-faces window texture.
- 2026-07-12 — storefront lighting is **varied** (main-street trait); department stores uniform bright white.
- 2026-07-12 — lab IDs: **hybrid sidecar** (not explicit-restructure, not content-hash).
