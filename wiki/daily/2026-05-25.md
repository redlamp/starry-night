#### Moon as a celestial body

- Old store fields `horizontalRadius / height / angleDeg` represented the moon as a cylinder-coord object floating over the city — that model couldn't capture "fixed point in the sky", so the "moon opp. cam" toggle kept drifting moon onto city center under different defaults
- Re-parametrised as `azimuthDeg / elevationDeg / distance` — a sky direction with a radial distance that defaults to `stars.radius` so the moon sits on the celestial sphere
- Toggle renamed `moonOppositeCamera` → `moonFollowCamera`; only overrides azimuth (camera azimuth + π), elevation and distance stay locked so the moon orbits at the same sky height + radius
- Defaults preserve the old visible position: `az 200° / el 32° / dist 4500`
- See [[decision-moon-celestial-model]]

#### Drag-to-spin orbit + jolt-free orbit speed

- `OrbitConfig` refactored from `{cameraY, startAngleDeg}` to spherical `{radius, azimuthDeg, elevationDeg}` — camera height + horizontal radius are derived, elevation 90° puts the camera directly above the city without flipping over
- Pointer / touch drag in orbit mode: horizontal = yaw, vertical = pitch clamped 0..90°, pinch + mouse wheel = zoom radius. Drag pauses auto-revolution and rebases `azimuthDeg + orbitStart` on release so there's no snap
- "Period" slider relabelled "orbit speed" and changes rebase azimuth + `orbitStart` to the current angle, fixing the bug where adjusting period made follow-cam moon jump
- See [[decision-orbit-spherical-controls]]

#### Debug panel overhaul

- Master debug visibility checkbox at top of Debug, persisted via `localStorage` key `starry-night.debugVisible` — new visitors land on a clean panel
- Orbit sliders moved out of the orbit-only block into Debug → 🌀 orbit subsection (reachable without engaging orbit mode); stars → ⭐, moon → 🌙 — Follow camera moved into the moon subsection header
- Save / Reset semantics: Save snapshots `cameraIntent + orbit + moon + stars` to `localStorage` key `starry-night.savedConfig`; Reset restores that snapshot and falls back to hardcoded defaults only when nothing has been saved
- Copy values button puts the same snapshot on the clipboard as JSON for pasting back into code
- See [[decision-debug-panel-architecture]]
