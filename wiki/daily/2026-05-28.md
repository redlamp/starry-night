## Shipped to main

- **Stars in ortho (#23)** — star field render in its own perspective pass with
  a private camera (later anchored at world origin = skybox; star camera fov
  locked to 60° regardless of projection so density matches between perspective
  and orthographic).
- **#16 aviation beacons** — instanced red points on skyscraper tops, blink
  driven by `(sharedTime, per-beacon phase)`. Emissive >1 for ACES bloom; size
  floor so beacons read from orbit range.
- **#22 lookAt precision** — round captured lookAt to 3dp at `setCameraIntent`
  + cleaned the DEFAULT_INTENT literal.
- **#20 hide Still** — removed Still from the user-facing mode toggle (Fly /
  Orbit only); kept Still internal for the headless capture script + pose
  presets per the PRD still-frame workflow.
- **#18 / #19 orbit Top-down + Default restore** — `tweenOrbitTopDown` tweens
  elevation→90, radius→4500, orthoSize→1000; pre-state captured into
  `orbitRestore`; Default preset enabled in orbit mode while a restore is set.
  Later refactored to a two-phase swing-arm GSAP timeline (angular swing first
  at constant radius, then radius + orthoSize zoom-out overlapping the late
  half) so the motion reads as a pivot around the city, not a diagonal stretch.
- **Hiccup fix on programmatic pause/resume** — `useFrame` now detects
  `orbitPaused` transitions in the same frame as the state change, baking the
  in-flight sweep into `orbit.azimuthDeg` and resetting `orbitStart` so the
  camera never snaps when the Top-down / Default tween toggles pause.
- **Gradual north-up at top-down** — `camera.up` interpolates from
  `(0,1,0)` toward `(0,0,1)` proportional to the elevation tween progress
  (`(currentEl - originalEl) / (90 - originalEl)`), so the screen-orientation
  change happens in lock-step with the tween instead of snapping at the end.
- **Settings UX (Tabs)** — "Tween to" and "Projection" both converted from
  button groups to shadcn Tabs (installed via `bunx shadcn@latest add tabs`).
  "Tween to" Tabs value reflects orbit preset state (`top-down` if
  `orbitRestore` is set, else `default`) so the active tab shows the current
  mode. Projection moved to sit directly below Tween to; FovOrSizeSlider moved
  directly below Projection.
- **Horizon + sky restructure**
  - Ground: plane → 1575m-radius disc (~2× city extent, 48 segments).
  - Star pass: removed `worldY<0` clip (lower hemisphere stars fill the strip
    between disc edge and geometric horizon).
  - Star camera anchored at world origin (skybox semantics) + locked at 60°
    fov so stars rotate purely with camera rotation (no parallax mismatch with
    the city look direction).
  - `StarPass` gained a `haze` slot + third render pass: `gl.render(starScene)`
    → `clearDepth` → `gl.render(hazeScene, mainCam)` → `clearDepth` →
    `gl.render(mainScene)`. Opaque ground/city overwrite haze where they cover.
  - Haze fragment shader: drop the lower fade — anything at `worldY <= uBottomY`
    is fully opaque haze colour, making the lower hemisphere a soft "virtual
    ground" out to the geometric horizon.
  - SkyGradient: new large back-side sphere painted before stars, vertical
    gradient from fog colour at horizon to a darker zenith tone — bridges the
    colour step between fogged ground at the disc edge and the sky above.
- **Moon size slider** — `moon.radiusRatio` field (default 0.0355) exposed as
  a `size` slider in the Moon panel.
- **Comprehensive Reset** — `resetCamera` no longer reads SavedConfig; it now
  factory-resets every panel-tunable field to its hardcoded default (extracted
  `DEFAULT_FOG`, `DEFAULT_HAZE`, `DEFAULT_CITY_PLANNING_VIS`, `DEFAULT_FLY_SPEED`,
  `DEFAULT_ORTHO_SIZE`, `DEFAULT_PROJECTION`). Runtime readouts preserved.
- **Tuned defaults** — fog `near 1280→2400`, haze defaults to user-tuned
  values (color `#1b2641`, topY 360, bottomY -15, intensity 1.1, radius 1450),
  orbit `lookAtY 150→222`, `azimuthDeg 108.215…→215.993…`, `elevationDeg
  7.5→6.25`.

- **#7 buildings face fronting roads** — front-row buildings (within
  ROAD_FACING_DIST=14m of a highway/arterial edge) snap rotationY to the
  road segment direction, but only when (a) the snap is within
  ROAD_FACING_ANGLE_TOL ≈ 28° of the block grid, and (b) the rotated
  footprint's perpendicular extent still fits the stripe's depth budget.
  Stride along the stripe uses the rotated footprint's projection on
  stripe-x so successors don't clip. Gate1 PASS after the perp-budget + tol
  + stride fixes (started at ~22 overlaps avg → 0). Steep-angle road
  crossings keep block rotation by design — preserves the stripe layout's
  invariants.
- **#25 unique windows per face** — `vFaceId` varying derived from
  BoxGeometry's local normal (+X=0, -X=1, +Z=2, -Z=3). Fragment shader
  shifts the atlas-sample cellId by `(faceId*7, faceId*11) mod gridXY`
  before computing atlasUv. Same atlas slice, same packer, same per-cell
  breathing/TV/wake animations — but each vertical face samples a different
  sub-pattern of the building's window state texture. Coprime offsets keep
  the 4 patterns distinct across typical grid sizes (3..20 cols/rows).
