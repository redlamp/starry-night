## Tensor-field city rebuild

Big day — the city generator was rebuilt from "slop" to a real streets-first tensor-field model, iteratively, with the user calibrating each step.

- Diagnosed the root cause: generator was **district-first**, not streets-first (blocks stamped over Voronoi bboxes; roads a separate, non-aligned pass → spider-web). Two adversarial design workflows (rebuild plan + tensor study of ProbableTrain/Parish-Müller).
- Shipped + then discarded an interim axis-aligned rectangular grid (continuous streets, rectangular blocks) once the user confirmed they wanted the organic tensor look.
- Built the tensor pipeline: `tensorField.ts` (grid + occasional radial basis fields, per-seed waviness), `tensorStreets.ts` (RK4 streamlines, two-phase deterministic seeding, both eigenvector families at two scales → criss-cross arterials + streets), `cityGen.ts` (ring→street demotion, longest-straight arterial → highway, road-frontage building fill).
- User feedback resolved across iterations: occasional (not ubiquitous) roundabouts; highways straight + spanning (never circular); arterials criss-cross + some streets promoted; denser streets; buildings line the streets, no orphan interiors.
- **Tensor is now the default + only intended model** (3D scene picks it up via plain seed); legacy + interim-grid bodies dormant pending purge.
- `/plan`: mode toggle removed, draw order districts < streets < arterials < highways < buildings, sorted layer menu, better fill icon.
- `gate1` rewritten to gate the tensor path: 20 seeds PASS (0 overlaps, 0 corridor hits, determinism), build green.
- Committed `fbc3d95` on `arch/city-planning`.
